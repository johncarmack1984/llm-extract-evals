# Fixtures (record / replay)

These JSON files are recorded model responses, keyed by a content hash of
`(model, run-index, input)`. They let `bun run eval` run **deterministically,
offline, with no API key and no spend** -- the eval replays a fixture when one
exists for the case, and the checked-in numbers are reproducible from the
checked-in fixtures.

## The committed fixtures are illustrative, not real recordings

The fixtures in this directory were generated by
`scripts/make-illustrative-fixtures.ts` from the gold labels in `../cases/`.
They are **synthetic stand-ins** that reproduce the documented example run (see
`../../README.md`): every case matches its label except `02-prairie-wind`,
which carries the deliberate `status: "active"` hallucination that exercises the
failure taxonomy. They are not captured from a live model.

## Recording real responses

With an API key set, capture real model output (overwriting the illustrative
fixtures for the current `MODEL`):

```sh
RECORD=1 bun run eval                 # record claude-opus-4-8 (default)
MODEL=claude-haiku-4-5 RECORD=1 bun run eval   # record a different model
```

Fixtures are keyed by model, so each model gets its own set. Re-running
`bun run scripts/make-illustrative-fixtures.ts` restores the synthetic ones.
